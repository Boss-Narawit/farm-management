# คู่มือติดตั้งระบบจัดการฟาร์ม
## ขั้นตอนการตั้งค่าทั้งหมด (ใช้เวลาประมาณ 30-45 นาที)

ระบบนี้ใช้ **Firebase Realtime Database** เป็น backend หลัก — ข้อมูลทุกอย่างซิงก์แบบ real-time ข้ามอุปกรณ์ทั้งหมดภายใน ~1 วินาที

---

## ขั้นตอนที่ 1 — สร้าง Firebase Project

1. ไปที่ https://console.firebase.google.com
2. คลิก **"Add project"** → ตั้งชื่อ เช่น `farm-management`
3. ปิด Google Analytics → คลิก **Create project**

### เปิดใช้ Realtime Database

4. เมนูซ้าย: **Build → Realtime Database**
5. คลิก **Create Database**
6. เลือก region: **Southeast Asia (asia-southeast1)**
7. เลือก **Start in test mode** → คลิก **Enable**

### เปิดใช้ Anonymous Auth

8. เมนูซ้าย: **Build → Authentication**
9. คลิก **Get started**
10. แท็บ **Sign-in method** → คลิก **Anonymous** → เปิดสวิตช์ Enable → **Save**

### ตั้ง Security Rules (สำคัญ)

11. กลับไปที่ **Realtime Database → Rules**
12. แทนที่ด้วย:
    ```json
    {
      "rules": {
        ".read": "auth != null",
        ".write": "auth != null"
      }
    }
    ```
13. คลิก **Publish**

### คัดลอก Firebase Config

14. เมนูซ้าย: คลิกไอคอน ⚙️ → **Project settings**
15. เลื่อนลงมาที่ **"Your apps"** → คลิกไอคอน **`</>`** (Web)
16. ตั้งชื่อ app: `farm-app` → คลิก **Register app**
17. คัดลอก `firebaseConfig` ที่แสดงขึ้นมา เก็บไว้ใช้ในขั้นตอนที่ 5

---

## ขั้นตอนที่ 2 — ตั้งค่า Google Apps Script (สำหรับ LINE Login)

Apps Script ทำหน้าที่แค่ **แลก LINE auth code → user profile** เท่านั้น (ไม่เกี่ยวกับข้อมูล log)

1. ไปที่ https://script.google.com → คลิก **New project**
2. ลบโค้ดเดิมออกทั้งหมด
3. คัดลอกโค้ดจากไฟล์ **google-apps-script.js** วางลงไป
4. แก้ไขค่าที่ด้านบน (ยังไม่ต้องใส่ LINE keys ตอนนี้ก็ได้):
   ```javascript
   const LINE_CLIENT_ID     = 'YOUR_LINE_CLIENT_ID';
   const LINE_CLIENT_SECRET = 'YOUR_LINE_CLIENT_SECRET';
   ```
5. คลิก **Deploy → New deployment**
6. Type: **Web app**
7. Execute as: **Me** | Who has access: **Anyone**
8. คลิก **Deploy** → กด Authorize → คัดลอก **Web App URL** เก็บไว้

---

## ขั้นตอนที่ 3 — สร้าง LINE Login Channel

1. ไปที่ https://developers.line.biz
2. สมัครหรือล็อกอินด้วยบัญชี LINE
3. คลิก **Create a new provider** → ตั้งชื่อ เช่น "FarmApp"
4. คลิก **Create a new channel** → เลือก **LINE Login**
5. กรอกข้อมูล:
   - Channel name: **ระบบจัดการฟาร์ม**
   - App type: **Web app**
6. หลังสร้างเสร็จ ไปที่แท็บ **Basic settings**
7. คัดลอก **Channel ID** (= LINE_CLIENT_ID) และ **Channel secret** (= LINE_CLIENT_SECRET)
8. ไปที่แท็บ **LINE Login** → ใส่ Callback URL:
   ```
   https://[ชื่อ-github-username].github.io/farm-app/
   ```
   (จะได้ URL จริงในขั้นตอนถัดไป)

---

## ขั้นตอนที่ 4 — อัพโหลด Web App (ฟรี ด้วย GitHub Pages)

1. สมัครบัญชีที่ https://github.com (ฟรี)
2. คลิก **New repository**
3. ตั้งชื่อ: **farm-app**
4. ติ๊ก **Public** → คลิก **Create repository**
5. อัพโหลดไฟล์ทั้งหมด (index.html, styles.css, js/, ฯลฯ)
6. ไปที่ **Settings → Pages**
7. Source: **Deploy from branch → main → / (root)**
8. รอสักครู่ จะได้ URL เช่น:
   ```
   https://[username].github.io/farm-app/
   ```
9. นำ URL นี้ไปใส่ใน LINE Developers Console (Callback URL)

---

## ขั้นตอนที่ 5 — แก้ไข CONFIG ในไฟล์ js/core/config.js

เปิดไฟล์ `js/core/config.js` และแก้ไขค่าทั้งหมด:

```javascript
const CFG = {
  LINE_CLIENT_ID: 'วาง Channel ID จาก LINE ตรงนี้',
  REDIRECT_URI: window.location.origin + window.location.pathname,
  APPS_SCRIPT_URL: 'วาง Web App URL จาก Apps Script ตรงนี้',
  FIREBASE: {
    apiKey: 'วางค่าจาก firebaseConfig ตรงนี้',
    authDomain: 'your-project.firebaseapp.com',
    databaseURL: 'https://your-project-default-rtdb.asia-southeast1.firebasedatabase.app',
    projectId: 'your-project'
  },
  ALLOWED_USER_IDS: [
    'Uxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx',  // LINE User ID ผู้จัดการคนที่ 1
    'Uxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx',  // LINE User ID ผู้จัดการคนที่ 2
  ]
};
```

แล้วอัพโหลดไฟล์ใหม่ไปที่ GitHub อีกครั้ง

## ขั้นตอนที่ 6 — หา LINE User ID ของผู้จัดการ

1. กลับไปที่ Apps Script editor → เพิ่มบรรทัดนี้ใน `handleLineAuth` ชั่วคราว:
   ```javascript
   Logger.log('User ID: ' + prof.userId);
   ```
2. ให้ผู้จัดการแต่ละคนล็อกอินครั้งแรก
3. ใน Apps Script คลิก **View → Logs** จะเห็น User ID รูปแบบ `Uxxxxxxxx...`
4. นำ User ID ไปใส่ใน `ALLOWED_USER_IDS` ในไฟล์ `js/core/config.js`
5. ลบบรรทัด Logger.log ออก แล้ว redeploy Apps Script
6. อัพโหลดไฟล์ config.js ที่แก้แล้วขึ้น GitHub

## ขั้นตอนที่ 7 — แจ้งผู้จัดการ

ส่ง URL นี้ให้ผู้จัดการทุกคน:
```
https://[username].github.io/farm-app/
```

แนะนำให้บันทึกลง **Home Screen** ของมือถือ:
- iPhone: เปิด Safari → กด Share → **Add to Home Screen**
- Android: เปิด Chrome → กด ⋮ → **Add to Home Screen**

จะได้ไอคอนบนหน้าจอเหมือนแอปปกติ — ข้อมูลซิงก์แบบ real-time ทุกอุปกรณ์อัตโนมัติ

## ปัญหาที่พบบ่อย (ระบบหลัก)

**Q: ล็อกอินแล้วขึ้น "ไม่ได้รับอนุญาต"**
A: LINE User ID ยังไม่ได้ถูกเพิ่มใน ALLOWED_USER_IDS ให้ทำขั้นตอนที่ 6

**Q: ข้อมูลไม่ซิงก์ข้ามอุปกรณ์**
A: ตรวจสอบว่า FIREBASE config ถูกต้องทุกค่า โดยเฉพาะ `databaseURL` และ Anonymous Auth เปิดอยู่

**Q: หน้าจอขึ้น Error หลัง LINE Login**
A: ตรวจสอบ Callback URL ใน LINE Developers Console ว่าตรงกับ URL ของเว็บ และ APPS_SCRIPT_URL ถูก Deploy แล้ว

**Q: ใช้งานได้ offline ไหม?**
A: ได้ — ข้อมูลจะถูก cache ไว้ในเครื่อง และซิงก์อัตโนมัติเมื่อมีอินเทอร์เน็ต

**Q: ต้องการดูข้อมูลในรูปแบบ Spreadsheet**
A: ใช้ปุ่ม "ดาวน์โหลดข้อมูลสำรอง" ได้ไฟล์ JSON แล้ว import เข้า Excel/Sheets เอง

---

*ข้อมูลทั้งหมดเก็บใน Firebase ของคุณเอง — ไม่มีบุคคลที่สามสามารถเข้าถึงได้*

# ส่วนที่ 2 — แชทบอท LINE (ถามข้อมูลฟาร์มผ่านแชท)

แชทบอทนี้ให้ผู้จัดการส่งข้อความถามข้อมูลฟาร์มในแชท LINE ได้โดยตรง เช่น:
- "วันนี้ใครทำงานบ้าง?"
- "สัปดาห์นี้ค่าจ้างรวมเท่าไหร่?"
- "แปลงไหนทำงานล่าสุด?"

บอทดึงข้อมูลจาก Firebase เดิมที่มีอยู่แล้ว — ไม่ต้องติดตั้งระบบหลักใหม่

**ต้องทำขั้นตอนที่ 1–6 ข้างบนให้เสร็จก่อน**

---

## ขั้นตอนที่ 8 — สมัคร Anthropic API Key (สำหรับ AI)

บอทใช้ AI ของ Anthropic (Claude) ในการเข้าใจคำถามภาษาไทยและตอบกลับ

1. ไปที่ https://console.anthropic.com
2. คลิก **Sign up** → สมัครด้วย Google หรืออีเมล
3. ยืนยันอีเมล → ล็อกอินเข้า Console
4. เมนูซ้าย: คลิก **API Keys**
5. คลิก **Create Key** → ตั้งชื่อ เช่น `farm-bot`
6. คัดลอก Key (ขึ้นต้นด้วย `sk-ant-...`) **เก็บไว้ใช้ในขั้นตอนที่ 11**

> ค่าใช้จ่าย: ประมาณ **$0.001 ต่อการสนทนา 10 ครั้ง** (ถูกมาก) มีเครดิตฟรีให้ทดลองใช้

---

## ขั้นตอนที่ 9 — สร้าง LINE Messaging API Channel (ช่องแชทบอท)

นี่คือ Channel ใหม่แยกต่างหาก ไม่ใช่ Channel LOGIN เดิม

1. ไปที่ https://developers.line.biz → ล็อกอิน
2. คลิก Provider เดิมที่สร้างไว้ (เช่น "FarmApp")
3. คลิก **Create a new channel** → เลือก **Messaging API**
4. กรอกข้อมูล:
   - Channel name: **ผู้ช่วยฟาร์มอ้อย**
   - Channel description: ระบบถามข้อมูลฟาร์มผ่านแชท
   - Category: Agriculture
5. ยอมรับข้อกำหนด → คลิก **Create**
6. ไปที่แท็บ **Basic settings** → คัดลอก **Channel secret** **เก็บไว้**
7. ไปที่แท็บ **Messaging API**:
   - เลื่อนลงมาที่ **Channel access token**
   - คลิก **Issue** → คัดลอก token ที่ได้ **เก็บไว้**

> ตอนนี้ยังไม่ต้องตั้ง Webhook — จะทำในขั้นตอนที่ 12

---

## ขั้นตอนที่ 10 — ดาวน์โหลด Firebase Service Account

บอทต้องการสิทธิ์ Admin เพื่ออ่านข้อมูลจาก Firebase โดยตรง (ไม่ผ่านเบราว์เซอร์)

1. ไปที่ https://console.firebase.google.com → เข้า Project เดิม
2. คลิกไอคอน ⚙️ (บนซ้าย) → **Project settings**
3. คลิกแท็บ **Service accounts**
4. คลิก **Generate new private key** → **Generate key**
5. ไฟล์ JSON จะถูกดาวน์โหลดอัตโนมัติ (เช่น `farm-management-firebase-adminsdk-xxx.json`)
6. เปิดไฟล์นั้นด้วย Notepad หรือ TextEdit → คัดลอก **ทั้งหมด** ตั้งแต่ `{` ถึง `}` **เก็บไว้**

> ห้ามแชร์ไฟล์นี้กับใคร — มีสิทธิ์เข้าถึงฐานข้อมูลทั้งหมด

---

## ขั้นตอนที่ 11 — Deploy บอทบน Railway (ฟรี)

Railway คือบริการ hosting ที่รัน bot server ให้ตลอดเวลา

### 11.1 สมัครและเชื่อม GitHub

1. ไปที่ https://railway.app → คลิก **Start a New Project**
2. เลือก **Sign in with GitHub** → อนุญาตการเข้าถึง

### 11.2 สร้าง Project ใหม่

3. คลิก **New Project** → เลือก **Deploy from GitHub repo**
4. เลือก repository: `farm-management`
5. Railway จะถามว่าจะ deploy ส่วนไหน — คลิก **Add variables** ก่อน (อย่าเพิ่ง Deploy)

### 11.3 ตั้งค่า Root Directory

6. ในหน้า Project settings → คลิกชื่อ Service → **Settings**
7. หัวข้อ **Source** → Root Directory: พิมพ์ `linebot`
8. คลิก **Save**

### 11.4 ใส่ Environment Variables

9. คลิกแท็บ **Variables** → คลิก **Add Variable** ทีละอัน:

   | Variable | ค่า |
   |----------|-----|
   | `LINE_CHANNEL_SECRET` | Channel secret จากขั้นตอนที่ 9 |
   | `LINE_CHANNEL_ACCESS_TOKEN` | Channel access token จากขั้นตอนที่ 9 |
   | `FIREBASE_SERVICE_ACCOUNT` | JSON ทั้งหมดจากขั้นตอนที่ 10 (วางทั้งก้อน) |
   | `FIREBASE_DATABASE_URL` | `https://YOUR_PROJECT_ID-default-rtdb.asia-southeast1.firebasedatabase.app` |
   | `ANTHROPIC_API_KEY` | API Key จากขั้นตอนที่ 8 |

   > **FIREBASE_DATABASE_URL**: ดูได้จาก Firebase Console → Realtime Database → Data → URL ที่แถบบน

10. คลิก **Deploy** → รอ 2-3 นาที

### 11.5 คัดลอก URL ของบอท

11. หลัง Deploy สำเร็จ คลิกแท็บ **Settings** → หัวข้อ **Networking**
12. คลิก **Generate Domain** → จะได้ URL เช่น:
    ```
    https://farm-management-production-xxxx.up.railway.app
    ```
13. **เก็บ URL นี้ไว้** ใช้ในขั้นตอนถัดไป

## ขั้นตอนที่ 12 — ตั้ง Webhook ใน LINE

1. กลับไปที่ https://developers.line.biz → เข้า Channel **ผู้ช่วยฟาร์มอ้อย**
2. คลิกแท็บ **Messaging API**
3. หัวข้อ **Webhook settings**:
   - Webhook URL: วาง URL จากขั้นตอนที่ 11 แล้วต่อท้ายด้วย `/webhook`
     ```
     https://farm-management-production-xxxx.up.railway.app/webhook
     ```
   - คลิก **Update**
4. เปิดสวิตช์ **Use webhook** → **ON**
5. คลิก **Verify** → ต้องขึ้น **Success**

> ถ้า Verify ไม่ผ่าน: รอ 1 นาทีแล้วลองใหม่ หรือตรวจสอบว่า Railway Deploy สำเร็จแล้ว

## ขั้นตอนที่ 13 — เพิ่มบอทในกลุ่ม LINE แล้วทดสอบ

1. ใน LINE Developers Console → แท็บ **Messaging API**
2. หัวข้อ **Bot information** → คลิก **Add bot as friend** หรือสแกน QR Code
3. เพิ่มบอทเข้ากลุ่มที่ผู้จัดการทุกคนอยู่ หรือแชทส่วนตัว
4. ส่งข้อความทดสอบ เช่น:
   - `วันนี้ใครทำงานบ้าง?`
   - `7 วันที่ผ่านมาค่าจ้างรวมเท่าไหร่?`
   - `มีพนักงานกี่คน?`

> บอทจะตอบภายใน 5-10 วินาที — ถ้าไม่ตอบให้ดู Railway Logs (แท็บ **Deployments → View Logs**)

## ปัญหาที่พบบ่อย (แชทบอท)

**Q: บอทไม่ตอบเลย**
A: ตรวจสอบตามลำดับ:
1. Railway → Deployments → ดู Logs ว่ามี Error ไหม
2. LINE → Messaging API → Verify Webhook → ต้องขึ้น Success
3. ตรวจสอบว่า Environment Variables ครบทุกตัวและถูกต้อง

**Q: Verify Webhook ขึ้น Error**
A: รอให้ Railway Deploy เสร็จสมบูรณ์ก่อน (ดูที่แถบ Status ใน Railway) แล้วลอง Verify ใหม่

**Q: บอทตอบว่า "ขออภัย เกิดข้อผิดพลาด"**
A: ดู Logs ใน Railway — มักเกิดจาก `FIREBASE_SERVICE_ACCOUNT` ไม่ถูกต้อง ให้ copy JSON ใหม่ทั้งก้อน

**Q: บอทตอบไม่ตรงกับข้อมูลในแอป**
A: บอทดึงข้อมูล 7 วันล่าสุดเท่านั้น ถ้าถามข้อมูลเก่ากว่านั้นจะไม่มีในระบบ

**Q: `FIREBASE_DATABASE_URL` หาได้จากไหน?**
A: Firebase Console → เลือก Project → Realtime Database → Data → ดู URL สีเทาด้านบนตาราง รูปแบบ `https://xxx-default-rtdb.asia-southeast1.firebasedatabase.app`
