# คู่มือติดตั้งระบบจัดการฟาร์ม
## ขั้นตอนการตั้งค่าทั้งหมด (ใช้เวลาประมาณ 30-45 นาที)

ระบบนี้ใช้ **Firebase Realtime Database** เป็น backend หลัก — ข้อมูลทุกอย่างซิงก์แบบ real-time ข้ามอุปกรณ์ทั้งหมดภายใน ~1 วินาที

---

## ขั้นตอนที่ 1 — สร้าง Firebase Project

1. ไปที่ https://console.firebase.google.com
2. คลิก **"Add project"** → ตั้งชื่อ เช่น `farm-management`
3. ปิด Google Analytics → คลิก **Create project**

### เปิดใช้ Realtime Database

4. เมนูซ้าย: **Build → Realtime Database**
5. คลิก **Create Database**
6. เลือก region: **Southeast Asia (asia-southeast1)**
7. เลือก **Start in test mode** → คลิก **Enable**

### เปิดใช้ Anonymous Auth

8. เมนูซ้าย: **Build → Authentication**
9. คลิก **Get started**
10. แท็บ **Sign-in method** → คลิก **Anonymous** → เปิดสวิตช์ Enable → **Save**

### ตั้ง Security Rules (สำคัญ)

11. กลับไปที่ **Realtime Database → Rules**
12. แทนที่ด้วย:
    ```json
    {
      "rules": {
        ".read": "auth != null",
        ".write": "auth != null"
      }
    }
    ```
13. คลิก **Publish**

### คัดลอก Firebase Config

14. เมนูซ้าย: คลิกไอคอน ⚙️ → **Project settings**
15. เลื่อนลงมาที่ **"Your apps"** → คลิกไอคอน **`</>`** (Web)
16. ตั้งชื่อ app: `farm-app` → คลิก **Register app**
17. คัดลอก `firebaseConfig` ที่แสดงขึ้นมา เก็บไว้ใช้ในขั้นตอนที่ 5

---

## ขั้นตอนที่ 2 — ตั้งค่า Google Apps Script (สำหรับ LINE Login)

Apps Script ทำหน้าที่แค่ **แลก LINE auth code → user profile** เท่านั้น (ไม่เกี่ยวกับข้อมูล log)

1. ไปที่ https://script.google.com → คลิก **New project**
2. ลบโค้ดเดิมออกทั้งหมด
3. คัดลอกโค้ดจากไฟล์ **google-apps-script.js** วางลงไป
4. แก้ไขค่าที่ด้านบน (ยังไม่ต้องใส่ LINE keys ตอนนี้ก็ได้):
   ```javascript
   const LINE_CLIENT_ID     = 'YOUR_LINE_CLIENT_ID';
   const LINE_CLIENT_SECRET = 'YOUR_LINE_CLIENT_SECRET';
   ```
5. คลิก **Deploy → New deployment**
6. Type: **Web app**
7. Execute as: **Me** | Who has access: **Anyone**
8. คลิก **Deploy** → กด Authorize → คัดลอก **Web App URL** เก็บไว้

---

## ขั้นตอนที่ 3 — สร้าง LINE Login Channel

1. ไปที่ https://developers.line.biz
2. สมัครหรือล็อกอินด้วยบัญชี LINE
3. คลิก **Create a new provider** → ตั้งชื่อ เช่น "FarmApp"
4. คลิก **Create a new channel** → เลือก **LINE Login**
5. กรอกข้อมูล:
   - Channel name: **ระบบจัดการฟาร์ม**
   - App type: **Web app**
6. หลังสร้างเสร็จ ไปที่แท็บ **Basic settings**
7. คัดลอก **Channel ID** (= LINE_CLIENT_ID) และ **Channel secret** (= LINE_CLIENT_SECRET)
8. ไปที่แท็บ **LINE Login** → ใส่ Callback URL:
   ```
   https://[ชื่อ-github-username].github.io/farm-app/
   ```
   (จะได้ URL จริงในขั้นตอนถัดไป)

---

## ขั้นตอนที่ 4 — อัพโหลด Web App (ฟรี ด้วย GitHub Pages)

1. สมัครบัญชีที่ https://github.com (ฟรี)
2. คลิก **New repository**
3. ตั้งชื่อ: **farm-app**
4. ติ๊ก **Public** → คลิก **Create repository**
5. อัพโหลดไฟล์ทั้งหมด (index.html, styles.css, js/, ฯลฯ)
6. ไปที่ **Settings → Pages**
7. Source: **Deploy from branch → main → / (root)**
8. รอสักครู่ จะได้ URL เช่น:
   ```
   https://[username].github.io/farm-app/
   ```
9. นำ URL นี้ไปใส่ใน LINE Developers Console (Callback URL)

---

## ขั้นตอนที่ 5 — แก้ไข CONFIG ในไฟล์ js/core/config.js

เปิดไฟล์ `js/core/config.js` และแก้ไขค่าทั้งหมด:

```javascript
const CFG = {
  LINE_CLIENT_ID: 'วาง Channel ID จาก LINE ตรงนี้',
  REDIRECT_URI: window.location.origin + window.location.pathname,
  APPS_SCRIPT_URL: 'วาง Web App URL จาก Apps Script ตรงนี้',
  FIREBASE: {
    apiKey: 'วางค่าจาก firebaseConfig ตรงนี้',
    authDomain: 'your-project.firebaseapp.com',
    databaseURL: 'https://your-project-default-rtdb.asia-southeast1.firebasedatabase.app',
    projectId: 'your-project'
  },
  ALLOWED_USER_IDS: [
    'Uxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx',  // LINE User ID ผู้จัดการคนที่ 1
    'Uxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx',  // LINE User ID ผู้จัดการคนที่ 2
  ]
};
```

แล้วอัพโหลดไฟล์ใหม่ไปที่ GitHub อีกครั้ง

## ขั้นตอนที่ 6 — หา LINE User ID ของผู้จัดการ

1. กลับไปที่ Apps Script editor → เพิ่มบรรทัดนี้ใน `handleLineAuth` ชั่วคราว:
   ```javascript
   Logger.log('User ID: ' + prof.userId);
   ```
2. ให้ผู้จัดการแต่ละคนล็อกอินครั้งแรก
3. ใน Apps Script คลิก **View → Logs** จะเห็น User ID รูปแบบ `Uxxxxxxxx...`
4. นำ User ID ไปใส่ใน `ALLOWED_USER_IDS` ในไฟล์ `js/core/config.js`
5. ลบบรรทัด Logger.log ออก แล้ว redeploy Apps Script
6. อัพโหลดไฟล์ config.js ที่แก้แล้วขึ้น GitHub

---

## ขั้นตอนที่ 7 — แจ้งผู้จัดการ

ส่ง URL นี้ให้ผู้จัดการทุกคน:
```
https://[username].github.io/farm-app/
```

แนะนำให้บันทึกลง **Home Screen** ของมือถือ:
- iPhone: เปิด Safari → กด Share → **Add to Home Screen**
- Android: เปิด Chrome → กด ⋮ → **Add to Home Screen**

จะได้ไอคอนบนหน้าจอเหมือนแอปปกติ — ข้อมูลซิงก์แบบ real-time ทุกอุปกรณ์อัตโนมัติ

## ปัญหาที่พบบ่อย

**Q: ล็อกอินแล้วขึ้น "ไม่ได้รับอนุญาต"**
A: LINE User ID ยังไม่ได้ถูกเพิ่มใน ALLOWED_USER_IDS ให้ทำขั้นตอนที่ 6

**Q: ข้อมูลไม่ซิงก์ข้ามอุปกรณ์**
A: ตรวจสอบว่า FIREBASE config ถูกต้องทุกค่า โดยเฉพาะ `databaseURL` และ Anonymous Auth เปิดอยู่

**Q: หน้าจอขึ้น Error หลัง LINE Login**
A: ตรวจสอบ Callback URL ใน LINE Developers Console ว่าตรงกับ URL ของเว็บ และ APPS_SCRIPT_URL ถูก Deploy แล้ว

**Q: ใช้งานได้ offline ไหม?**
A: ได้ — ข้อมูลจะถูก cache ไว้ในเครื่อง และซิงก์อัตโนมัติเมื่อมีอินเทอร์เน็ต

**Q: ต้องการดูข้อมูลในรูปแบบ Spreadsheet**
A: ใช้ปุ่ม "ดาวน์โหลดข้อมูลสำรอง" ได้ไฟล์ JSON แล้ว import เข้า Excel/Sheets เอง
